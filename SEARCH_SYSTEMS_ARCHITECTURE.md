## 2. SEARCH MEMORY STRATEGY

**Goal:** Make Google "remember" the site as a stable reference point.

### **2.1 URL Stability (The Memory Anchor)**
**Rule:** URL changes erase memory. 
- **Policy:** Never change a URL once indexed.
- **Legacy:** If a product is discontinued, the URL stays 200 OK and explains it (or redirects 301). Never 404 a URL with history.

### **2.2 ID Consistency**
We use persistent `@id` nodes in Schema to link data across pages.
- **Organization Node:** `https://www.skrmetalsuppliers.in/#organization`
- **Website Node:** `https://www.skrmetalsuppliers.in/#website`
- **Effect:** Every page reinforces the *same* organization entity, compounding trust rather than fragmenting it.

### **2.3 The "Reference" Content Tier**
We position your Technical Guides not as "blog posts" (ephemeral) but as **"Reference Material"** (stable).
- **Signal:** `Article` schema, updating `dateModified`, stable URLs.
- **Outcome:** Google treats these as answers, not just news.

## 3. AUTHORITY WITHOUT CLAIMS ("Quiet Authority")

**Goal:** Earn trust through structural correctness, not marketing hype.

### **3.1 Structural Precision**
Instead of saying "We are the best," we *prove* structure:
- **Clean Sitemaps:** Only high-value pages.
- **Valid HTML5:** Perfect semantic nesting.
- **Fast Core Web Vitals:** Respect for user time.
- **Zero Broken Links:** Maintenance discipline.

**Google's logic:** "This site is maintained with corporate-grade discipline; it is likely a trustworthy entity."

### **3.2 The Zero-Noise Policy**
We do not let Google index junk.
- **Blocked:** API routes, admin paths, tracking parameters.
- **Noindexed:** Thin pages, duplicate collections.
- **Result:** Every indexed page is high-value. This increases your site's **"Average Quality Score"** in the index.

## 4. TEMPORAL SEO (Time as a Signal)

**Goal:** Use time to verify legitimacy.

### **4.1 The "Stability" Signal**
- **Core Pages (Home, About, Products):** Change *rarely*. Stability = Institutional Trust.
- **Update Rhythm:** Minor updates every 6-12 months to verify business is active (`dateModified`).

### **4.2 The "Freshness" Signal**
- **Guide Pages:** Update significantly every 3-6 months.
- **Why:** Signals you are actively maintaining technical accuracy.

### **4.3 Domain Age Compounding**
- We are building a foundation that gets *stronger* simply by existing without errors. 
- **Strategy:** Avoid drastic backend changes. Let the domain age cleanly.
